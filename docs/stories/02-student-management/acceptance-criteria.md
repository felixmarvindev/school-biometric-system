# Story 02: Student Management - Acceptance Criteria

## Story-Level Acceptance Criteria

- [ ] Students can be created with all required information
- [ ] Admission numbers are unique per school
- [ ] Students can be assigned to classes and streams
- [ ] Student list displays all students with filtering
- [ ] Student information can be viewed and updated
- [ ] Students can be deactivated (soft delete)
- [ ] Parent contact information is stored and validated

## Phase-Level Criteria

### Phase 1: Student Data Model
- [ ] Student table created with all fields
- [ ] Class and Stream tables created
- [ ] Relationships defined correctly
- [ ] Foreign keys and constraints work

### Phase 2: CRUD Operations
- [ ] All CRUD endpoints work correctly
- [ ] Validation prevents invalid data
- [ ] Pagination works for large lists
- [ ] Filtering by name works

### Phase 3: Class Assignment
- [ ] Classes and streams can be created
- [ ] Students can be assigned to class/stream
- [ ] Filtering by class/stream works
- [ ] Class/stream displays in student list

### Phase 4: Parent Contacts
- [ ] Parent phone and email stored
- [ ] Phone format validated
- [ ] Email format validated
- [ ] Parent info displays correctly

---

**All criteria must be met before marking story complete.**
